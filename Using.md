# 个人复习题库 - 使用说明

## 一、项目简介

个人复习题库是一款基于 React + FastAPI 构建的本地刷题学习工具，支持题库管理、刷题练习、错题本、AI 解题、AI 学习监督等功能，帮助你高效复习和巩固知识。

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React 18 + Vite + Tailwind CSS + Zustand |
| 后端 | FastAPI + SQLAlchemy + SQLite |
| AI | YOLOv8 + YOLOv8-pose + MediaPipe + OpenAI 兼容 API |

---

## 二、功能介绍

### 1. 首页概览
- 今日学习数据展示（答题数、正确率、学习时长）
- 近 7 天刷题趋势图
- 各分类掌握度环形图
- 待复习错题数、收藏题目数
- 快捷入口（开始刷题、错题复习、题库管理）

### 2. AI 解题
- **智能问答**：兼容 OpenAI 格式的 API，支持流式输出
- **从题库导入**：一键从题库选择题目导入到输入框
- **分类筛选**：按分类快速筛选题目
- **演示模式**：未配置 API 时自动启用，模拟完整解题过程
  - 题目分析 → 解题思路 → 详细解答 → 总结拓展
  - 支持 TCP/UDP、排序算法、死锁、二分查找等智能匹配
- **快捷提问**：预设常用问题一键发送
- **配置管理**：在系统设置中配置 API Key、模型、地址等

### 3. 题库管理
- **题目 CRUD**：新增、编辑、删除、复制题目
- **批量操作**：批量删除题目
- **筛选搜索**：按分类、题型、难度、关键词搜索
- **多题型支持**：单选题、多选题、判断题、简答题
- **标签系统**：为题目添加多个标签
- **难度分级**：简单、中等、困难三级难度

### 4. 数据导入导出
- **JSON 导入/导出**：支持 JSON 格式的题库备份
- **Excel (xlsx) 导入/导出**：支持 Excel 表格批量导入导出
- **下载模板**：提供 Excel 导入模板
- **Markdown 导出**：导出为 Markdown 格式方便阅读

### 5. 刷题练习
- **智能出题**：按分类、题型、难度筛选题目
- **答题界面**：上一题/下一题、答题卡、进度显示
- **即时反馈**：提交后立即显示对错和解析
- **学习记录**：自动记录答题历史

### 6. 错题本
- **自动收录**：答错的题目自动加入错题本
- **待复习/已掌握**：分类管理错题状态
- **标记掌握**：复习做对后可标记为已掌握
- **错题统计**：查看各分类错题数量

### 7. 收藏夹
- **多收藏夹**：创建多个收藏文件夹分类管理
- **收藏/取消**：一键收藏或取消收藏题目
- **文件夹管理**：新建、编辑、删除收藏夹

### 8. 分类标签管理
- **分类管理**：创建、编辑、删除题目分类
- **标签管理**：创建、编辑、删除标签
- **颜色自定义**：为标签设置不同颜色

### 9. 数据统计
- **刷题趋势**：近 7 天/30 天答题趋势图
- **分类掌握度**：各分类的正确率对比
- **错题分析**：错题分类分布
- **学习时长**：累计学习时间统计
- **专注度记录**：AI 监督的专注度历史

### 10. AI 学习监督
- **实时摄像头监督**：通过摄像头实时检测学习状态
- **6 种分心行为检测**：
  - 📱 玩手机（YOLO 物品检测）
  - 🍔 吃东西（YOLO 物品检测）
  - 😴 趴桌睡觉（YOLOv8-pose 姿态估计）
  - 😵 眼睛闭上（OpenCV 级联分类器）
  - 🤦 撑脸（YOLOv8-pose 姿态估计）
  - 🚶 离开座位（YOLO 人物检测）
- **专注度评分**：初始 100 分，每种分心行为持续 5 秒扣 1 分
- **悬浮摄像头小球**：可拖拽移动，右键菜单操作
- **实时监督窗口**：查看检测画面、骨架关键点、评分
- **评分规则折叠**：底部可折叠的评分规则说明

### 11. 系统设置
- **主题切换**：亮色/暗色主题
- **悬浮球设置**：显示/隐藏悬浮摄像头小球
- **AI 解题配置**：API Key、模型、地址、温度、系统提示词
- **数据管理**：JSON/Excel 导入导出、下载模板
- **关于信息**：版本号等信息

## 三、启动项目

### 环境要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Python | ≥ 3.10 | 后端运行环境 |
| Node.js | ≥ 16.0 | 前端构建工具 |
| npm | ≥ 8.0 | 包管理器（Node.js 自带） |

---

### 第一步：启动后端

#### 1. 进入后端目录
```bash
cd backend
```

#### 2. 创建虚拟环境（推荐）
```bash
py -m venv venv
```

Windows 激活虚拟环境：
```bash
venv\Scripts\activate
```

#### 3. 安装依赖
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

> 使用清华源加速下载，国内网络推荐使用。

#### 4. 初始化数据（首次运行）
```bash
py init_data.py
```

> 这会创建 SQLite 数据库和初始分类。

#### 5. 启动后端服务
```bash
py -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

或者直接双击运行：
```
backend/start.bat
```

启动成功后，在浏览器访问 http://localhost:8000/docs 可以查看 API 文档。

---

### 第二步：启动前端

#### 1. 进入前端目录
```bash
cd frontend
```

#### 2. 安装依赖
```bash
npm install
```

如果下载慢，可以使用淘宝镜像：
```bash
npm install --registry=https://registry.npmmirror.com
```

#### 3. 启动开发服务器
```bash
npm run dev
```

或者直接双击运行：
```
frontend/start.bat
```

#### 4. 访问项目
启动成功后，在浏览器打开：
```
http://localhost:5173
```

---

### 快速启动（Windows）

如果你已经安装好环境，只需两步：

1. 双击 `backend/start.bat` 启动后端
2. 双击 `frontend/start.bat` 启动前端
3. 浏览器打开 http://localhost:5173

## 四、AI 功能配置

### 1. AI 解题配置

1. 打开系统设置 → AI 解题设置
2. 填入以下信息：
   - **API Key**：你的 API 密钥（如 sk-xxx）
   - **API 地址**：兼容 OpenAI 格式的 API 地址
     - OpenAI 官方：`https://api.openai.com/v1`
     - DeepSeek：`https://api.deepseek.com/v1`
     - 智谱 AI：`https://open.bigmodel.cn/api/paas/v4`
     - Moonshot：`https://api.moonshot.cn/v1`
   - **模型名称**：如 `gpt-3.5-turbo`、`deepseek-chat` 等
   - **温度**：0-2 之间，越高越随机，越低越确定
   - **系统提示词**：AI 的身份设定
3. 点击「保存配置」

> 未配置 API 时会自动进入演示模式，可体验完整的解题交互流程。

### 2. AI 学习监督

AI 监督功能开箱即用，无需额外配置：

- 首次使用会自动下载 YOLO 模型（约 6MB + 8MB）
- 确保电脑有可用的摄像头
- 浏览器会请求摄像头权限，请点击允许

## 五、题库导入说明

### Excel 导入格式

| 题型 | 题目 | 选项A | 选项B | 选项C | 选项D | 答案 | 解析 | 难度 | 分类 | 标签 |
|------|------|-------|-------|-------|-------|------|------|------|------|------|

#### 字段说明

| 字段 | 说明 | 必填 |
|------|------|------|
| 题型 | single(单选) / multiple(多选) / judge(判断) / short_answer(简答) | ✅ |
| 题目 | 题目内容，支持 Markdown | ✅ |
| 选项A-D | 选项内容，判断/简答题可留空 | 单选/多选必填 |
| 答案 | 单选：A，多选：AB，判断：true/false | ✅ |
| 解析 | 答案解析 | 否 |
| 难度 | easy / medium / hard | 否 |
| 分类 | 分类名称，不存在自动创建 | 否 |
| 标签 | 多个标签用英文逗号分隔 | 否 |

### JSON 导入格式

```json
[
  {
    "question_type": "single",
    "title": "题目内容",
    "options": ["A. 选项1", "B. 选项2"],
    "answer": "A",
    "analysis": "解析",
    "difficulty": "medium",
    "category_name": "分类名",
    "tags": ["标签1", "标签2"]
  }
]
```

## 六、常见问题

### Q: 摄像头无法开启？
A: 检查以下几点：
- 摄像头是否被其他应用占用
- 浏览器是否允许了摄像头权限
- 设备管理器中摄像头是否正常工作

### Q: AI 解题没有反应？
A: 
- 检查是否配置了 API Key
- 检查 API 地址和模型名称是否正确
- 未配置 API 时为演示模式，答案是预设的

### Q: 端口被占用怎么办？
A: 
- 后端：修改启动命令的 `--port` 参数
- 前端：修改 `frontend/vite.config.js` 中的 server.port
- 前端代理：修改 `frontend/vite.config.js` 中的代理目标端口

### Q: 数据保存在哪里？
A: 所有数据保存在 `backend/data/question_bank.db`（SQLite 数据库文件），备份时直接复制这个文件即可。

### Q: 如何重置数据？
A: 删除 `backend/data/question_bank.db` 文件，然后重新运行 `py init_data.py`。
